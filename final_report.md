# 表情包偷取与识别插件

已完成独立 AstrBot 插件实现：

- 群聊图片监听与白名单控制；
- 视觉模型识别 + 情景模型分类；
- 兼容 meme_manager 的 `memes/<分类>/` 与 `memes_data.json` 数据契约；
- SHA-256 去重、大小限制、并发限制和模型失败降级；
- meme_manager 依赖健康检查、定时复查和 `/表情偷取状态` 诊断命令；
- 完成配置 schema、README、单元测试和代码复核。
- 增加 `/偷取 [图片]` 即时入口；同一事件不会被自动监听重复处理，且已存在的图片会在调用模型前直接跳过。
- 通过发送前高优先级钩子清理 meme_manager 的 `&&category&&` 标记，并由本插件独立执行情景判断、分类取图、发送概率和会话冷却。
- `vision_provider_id` 可指定支持图片输入的视觉 Provider；`scene_provider_id` 负责偷取分类，`reply_scene_provider_id` 可单独指定智能回复表情模型，未填写时复用前者。
- 表情库索引改为 meme_manager 健康后自动后台扫描：目录名作为权威分类，只在原目录内补充多模态索引、编号和 `README.md`，不再提供手动整理命令，也不会跨目录移动图片。
- 针对 Gemini 按次计费，后台索引支持批量图片请求，自动回复表情合并为单次“是否发送+分类+选图”请求。
- 同一条消息的多张新图片改为一次批量视觉识别和一次批量情景分类，接口异常时才回退逐张调用。

安装后需要将 `metadata.yaml` 中的作者和仓库地址改成实际信息，并在 AstrBot 配置页填写支持图片的视觉 Provider。

使用 `/偷取 [图片]` 可立即处理同一条消息中的图片；自动发送由本插件的发送前钩子统一接管。当前本地测试 17 项全部通过。
